26// Pages are the central entity within Ikarus and allow you to describe things in
27// your world. Each page is associated with a list of properties and holds a value
28// for each of them.
29//
30// Pages can be derived from a template in which case the properties within that
31// template are synced to it.
32//
33// Templates can be removed from or added to existing pages in which case their
34// properties will also be removed from or added to that page. TODO
35//
36// Examples for Pages would be "Lord Balthazar III", "The Rusty Tankard" or "Wooden
37// Spoon".
38//
39// Not every wooden object is a piece of furniture (a wooden bridge isn't). Not
40// every piece of furniture is a wooden object (a safe isn't). So inheritance
41// doesn't address this issue properly. But a wooden table would still be both a
42// piece of furniture and a wooden object.
43//
44// Pages used to be called "Instances". They have renamed because of a planned
45// shift in Ikarus' user-interface where instances can be edited similar to
46// wiki-pages. In that sense "Page" seems more appropriate.
